# Codex Rules SSOT

This directory is the SSOT for Codex execution-policy rules.

## Files

- `base.rules`: universal baseline safety + common Kami Flow sync allowances.
- `profiles/dogfood.rules`: dogfood-friendly overlay.
- `profiles/client.rules`: client-project-safe overlay.

At sync time, runtime `kamiflow.rules` is generated by composing:

- `base.rules`
- `profiles/<selected-profile>.rules`

Flow command policy defaults:

- `kfc flow ensure-plan`: allow
- `kfc flow ready`: allow
- `kfc flow next`: allow
- `kfc flow apply`: allow in base/dogfood/client profiles

## Runtime Outputs (Generated)

These outputs are generated by sync scripts and should not be manually edited:

- `<repo>/.codex/rules/kamiflow.rules`
- `<project>/.codex/rules/kamiflow.rules`
- `$CODEX_HOME/rules/kamiflow.rules` (or `~/.codex/rules/kamiflow.rules`)

Profile selection precedence:

1. `--profile <dogfood|client>`
2. `<project>/kamiflow.config.json` -> `codex.rulesProfile`
3. default `client`

## Sync Commands

```bash
# All resources (skills + rules; default scopes)
npm run codex:sync

# Rules only
npm run codex:sync:rules -- --scope repo --force
npm run codex:sync:rules -- --scope project --project <path-to-project> --force
npm run codex:sync:rules -- --scope home --force

# Profile convenience
npm run codex:sync:rules:dogfood -- --scope project --project <path-to-project> --force
npm run codex:sync:rules:client -- --scope project --project <path-to-project> --force
```

Do not overwrite `.codex/rules/default.rules` from SSOT. Keep it for Codex-managed approvals.
