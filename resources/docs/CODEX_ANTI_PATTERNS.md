# Codex Anti-Patterns (SSOT)

Use this catalog to convert repeated mistakes into deterministic guardrails.

| ID | Scope | Bad Pattern | Detection Signal | Corrective Command | Rule Target | Skill Target |
| --- | --- | --- | --- | --- | --- | --- |
| AP-001 | client | Run repo-only `npm run ...` commands in client project | command boundary violation, wrong-context scripts (`dogfood:*`, `codex:sync*`, release scripts) | `kfc client` (or `kfc client bootstrap --project . --profile client`) | `resources/rules/profiles/client.rules` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-002 | client | Use direct `kfc-plan` guidance for client-facing workflows | docs or output suggests `kfc-plan` where `kfc` is required | `kfc client` then continue with `kfc flow ...` commands | `resources/rules/profiles/client.rules` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-003 | runtime | Manually edit generated runtime files (`.agents/skills`, `.codex/rules/kamiflow.rules`) | manual edits in generated paths, drift from SSOT | `npm run codex:sync -- --scope repo --force` (repo) or `kfc client --force` (client) | `resources/rules/base.rules` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-004 | runtime | Incorrect Codex CLI invocation/quoting (`spawn ENOENT`, `unexpected argument`) | `spawn codex ENOENT`, `unexpected argument ...` | Use validated wrapper command path and quoted prompt via KFC flow automation (for manual fallback: `codex exec \"<prompt>\"`) | `resources/rules/base.rules` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-005 | git | Commit blocked by hook crash (`env.exe` Win32 error 5) | `couldn't create signal pipe, Win32 error 5` | `git commit --no-verify -m \"<message>\"` then record fallback reason in task summary | `resources/rules/base.rules` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-006 | plan | Plan bootstrap command fails and flow stalls | `kfc plan init ... --new` failure, no active plan file | `kfc flow ensure-plan --project .` (fallback: `kfc plan init --project . --new`) | `resources/rules/base.rules` | `resources/skills/kamiflow-core/references/plan.md` |
| AP-007 | plan | Skip readiness gate before implementation | `build`/`fix` starts without proving plan build-readiness in markdown | run `$kamiflow-core plan` to satisfy gates (`decision`, handoff, checklist readiness), then rerun build/fix | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/skills/kamiflow-core/references/build.md`, `resources/skills/kamiflow-core/references/fix.md` |
| AP-008 | plan | Respond without mutating plan state | response includes route result but `.local/plans/*.md` unchanged | mutate frontmatter + `WIP Log` before final response | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/skills/kamiflow-core/SKILL.md` |
| AP-009 | plan | Report PASS but leave active plan unarchived | `check` says PASS while plan remains non-done and not moved to `.local/plans/done/` | apply archive gate and move plan to `.local/plans/done/` when criteria/gates are fully checked | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/skills/kamiflow-core/references/check.md` |
| AP-010 | quality | Claim outcomes without evidence (hallucinated status) | statements of success/failure lack command output or file evidence | mark claim as `Unknown`, run validation or route to `research` before asserting | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/skills/kamiflow-core/SKILL.md`, `resources/skills/kamiflow-core/references/build.md`, `resources/skills/kamiflow-core/references/fix.md` |
| AP-011 | plan | Create too many plan files for follow-up requests | multiple same-day plans with no scope split; active plan left unused | reuse active non-done plan; create new file only when no active plan exists or explicit scope split | `scripts/policy/verify-codex-intelligence-contract.mjs` | `AGENTS.md`, `resources/skills/kamiflow-core/SKILL.md`, `resources/skills/kamiflow-core/references/plan.md` |
| AP-012 | plan | Touch plan only once in a top-level request | missing route-start or pre-final plan mutation evidence (`updated_at` + `WIP Log`) | mutate active plan at route start and again before final response | `scripts/policy/verify-command-boundary.mjs` | `AGENTS.md`, `resources/skills/kamiflow-core/SKILL.md` |
| AP-013 | check | Mark done when completion is below 100% | `check` result indicates done/archive while checklist completion is incomplete | set `decision: NO_GO`, amend tasks/criteria, and continue `build/fix -> check` | `scripts/policy/verify-command-boundary.mjs` | `resources/skills/kamiflow-core/references/check.md` |
| AP-014 | routing | Execute low-confidence route instead of rerouting | selected route continues with `Route Confidence <4` or no confidence value present | return `Status: REROUTE`, choose fallback route (`start|plan|research`), persist reason, then re-run on fallback route | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/skills/kamiflow-core/SKILL.md`, `resources/skills/kamiflow-core/references/command-map.md` |
| AP-015 | routing | Misuse the no-plan fast path for implementation-bearing work | feature/refactor/UI/spec task runs without plan lifecycle or plan touch evidence | stop fast-path execution, resolve/reuse active non-done plan, then continue through normal route flow | `scripts/policy/verify-codex-intelligence-contract.mjs` | `AGENTS.md`, `resources/skills/kamiflow-core/SKILL.md`, `resources/docs/CODEX_FLOW_SMOOTH_GUIDE.md` |
| AP-016 | client | Store private client lessons in tracked docs or leave private runtime folders out of `.gitignore` | client-specific memory appears in Git changes or bootstrap omits `.kfc/`, `.local/`, `.agents/` ignore entries | keep curated client memory in `.kfc/LESSONS.md`, raw history in `.local/kfc-lessons/`, and rerun `kfc client --force` to restore private scaffolding | `scripts/policy/verify-codex-intelligence-contract.mjs` | `resources/docs/QUICKSTART.md`, `resources/docs/CLIENT_A2Z_PLAYBOOK.md`, `src/commands/client.ts` |
